Why?

Creating clean architecture Android app was always pain. RxFuel makes it simple with the power of RxJava and Kotlin. Manage your data flow with a single Observable stream and renders the UI with view states.

Clean Code

RxFuel makes your activity highly decoupled with business logic. All events thats happens in the activity is converted as an Observable and receives a view state after execution to render the UI.

From the events obervable to receiving the final view state, RxFuel handles it in a single observable stream. Making it clean with no spaghetti code.

High Testability

Unit testing your code is much easier with RxFuel. All events happening in the activity is emitted as an event object. In unit test, you can just emit a event object to assert an expected view state is observed.

RxFuel reduces the need of writing lots of instrumentation test as all events can be tested in unit test itself.

View state restoration

Everytime an activity is recreated due to a config change such as screen rotation or app restoration to foreground, the last view state is re-emitted to render the UI to latest state. You do not need to handle any state restoration manually.

This actually eliminates the need of using fragments to restore the state. Making the code much simpler.

This framework is in alpha stage and not recommended for production.
